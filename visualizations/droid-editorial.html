The Cascade

How a single useMemo brought down a process

Het begint onschuldig: Header.tsx gebruikt useMemo om een git readiness hint te berekenen. Maar getReadinessHint roept statSync aan — synchronis I/O tijdens de render cycle.

Bij elke notification (31,994×) re-rendert de Header. Elke render doet 4-5 statSync calls door de directory tree. De event loop blokkeert. CPU piekt naar 56%.

React probeert te reconcileren maar treft duplicate keys (70×) — het gooit de subtree weg en herbouwt deze. Een full re-render die alles opnieuw triggert. De maximum update depth (8×) is het vangnet dat de oneindige loop breekt.
