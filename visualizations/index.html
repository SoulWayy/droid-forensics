Forensics

The TUI renders at 6-32fps instead of the target 60fps. The main thread blocks on synchronous filesystem calls during React's render cycle.

Verdict

Sync I/O in the React render path

Header.tsx calls getReadinessHint() inside a useMemo during render. This triggers statSync() on every parent directory — synchronous filesystem I/O that blocks the JavaScript event loop on every notification.

// The problem:
const readiness = useMemo(() => {
return getReadinessHint(cwd); // ← sync statSync!
}, [cwd]);
// The fix:
const [readiness, setReadiness] = useState(null);
useEffect(() => {
getReadinessHintAsync(cwd).then(setReadiness); // ← async
}, [cwd]);

3-Layer Binary

Architecture

Droid is a single 150MB ELF binary with three layers: Rust core, JavaScriptCore, and a 53MB JS bundle.

Layer 1 — Rust Core

~50MB .text • 523 crates

Bun canary v1.3.14 (Rust rewrite). Async runtime, HTTP/TLS, PTY, JSON-RPC.

tokio 1.49 hyper 1.8 reqwest 0.12 rustls 0.23 portable-pty 0.8 ring 0.17

Layer 2 — JavaScriptCore

WebKit JSC • clang 21.1.5

JS engine with JIT compiler and garbage collector. NAPI bindings to Rust.

JIT Worker thread GC HeapHelper × 3 74GB VSZ (normal) Bun Pool × 4

Layer 3 — JS Bundle

53MB .bun (BunFS)

Ink/React TUI, agent logic, plugins, commands. Embedded as BunFS filesystem.
