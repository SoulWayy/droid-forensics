Verdict

De root cause is synchrone I/O in de React render path. Header.tsx roept statSync aan in een useMemo tijdens render, wat de event loop blokkeert. Render latency piekt naar 168ms (10× te lang voor 60fps).

Root Cause
Header.tsx gebruikt useMemo om getReadinessHint te berekenen. Dit roept statSync aan om .git te vinden in elke parent directory — synchronis I/O tijdens render.
Bij elke notification (31,994×) re-rendert de Header. Elke render doet 4-5 statSync calls. De event loop blokkeert. Render latency piekt naar 168ms (10× te lang voor 60fps). CPU naar 56%.
Fix
Verplaats getReadinessHint naar een useEffect met async I/O. Gebruik useState om het resultaat op te slaan.
